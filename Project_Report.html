APPENDIX A – IMPLEMENTATION

A.1 Terraform Infrastructure

Terraform is utilized for Infrastructure as Code (IaC) to manage AWS resources including VPC, Subnets, EKS Clusters, and ECR Repositories. Figure A.1 illustrates the cluster and node group definition.

# infra/terraform/aws/main.tf resource "aws_eks_cluster" "cppe" { name = var.cluster_name role_arn = aws_iam_role.eks_cluster_role.arn vpc_config { subnet_ids = aws_subnet.private[*].id } } resource "aws_eks_node_group" "main" { cluster_name = aws_eks_cluster.cppe.name node_group_name = "main-node-group" node_role_arn = aws_iam_role.eks_node_role.arn scaling_config { desired_size = 2 max_size = 3 min_size = 1 } }
Bottom: Figure A.1 Terraform Provisioning for AWS EKS

A.2 Docker Containerization

The application is containerized to ensure consistency across local development and production EKS environments. A docker-compose file orchestrates the app alongside monitoring tools.

# docker-compose.yml services: app: build: . ports: ["3000:3000"] environment: - OLLAMA_ENABLED=true - RCA_API_URL=${RCA_API_URL} prometheus: image: prom/prometheus volumes: ["./monitoring/prometheus/prometheus.yml:/etc/prometheus/prometheus.yml"] ports: ["9090:9090"]
Bottom: Figure A.2 Docker-Compose Orchestration Snippet
